2025년 1월부터 4월까지 진행된 보안 강화 프로젝트는 다수의 패스워드 교체와 경로 재정비를 수반했으며, 일부 회의록에는 노출되어서는 안 될 기밀 정보가 부주의하게 기록되기도 했다. 1월 8일 회의에서는 기존 VPN 서버 인증용 패스워드가 너무 단순하다는 지적이 나왔고, 이에 따라 네트워크 엔지니어 김하늘이 VPN 관리자 계정 비밀번호를 기존 'admin1234'에서 'V!pn#2025$Secure'로 변경하고, 설정 파일 '/etc/openvpn/server.conf' 경로에 직접 반영하였다. 이 사실은 회의록에 수정 내역과 함께 기록되었는데, 문제는 해당 파일 경로와 신규 비밀번호가 함께 기재되어 노출 리스크가 발생했다는 점이다. 또한 1월 22일 회의에서는 사내 주요 파일 서버의 접속 계정 비밀번호 변경도 결정되었는데, 시스템 관리자 최은지가 'fileserver01'의 관리자 계정 패스워드를 'Chang3Me!@2025'로 교체하고, Samba 공유 경로인 '\\internal-server\securefiles'에 이를 적용했다는 내역이 자세히 기록됐다. 이 변경 또한 파일 경로와 새로운 비밀번호가 노출된 형태로 회의록에 남았다. 2월 5일 회의에서는 클라우드 스토리지 접속을 위한 AWS IAM 사용자 비밀번호를 변경했는데, 보안 엔지니어 서민재가 직접 AWS Console에서 'aws_admin' 계정의 패스워드를 'AWS!Acc3ss$2025'로 갱신하고 S3 버킷 경로 's3://internal-documents-2025/'를 새롭게 등록했다는 내용이 일일이 문서에 담겼다. 이어진 2월 10일 회의에서는 Splunk 대시보드 구축을 위해 splunk_admin 계정 비밀번호를 'Splunk#Data!2025'로 변경하고, 설정 파일 '/opt/splunk/etc/system/local/authentication.conf'에 적용한 작업 내용이 상세히 공유됐다. 더욱 심각한 것은 3월 3일 클라우드 접근 통제 회의 때, 데이터 아키텍트 송지후가 주요 데이터베이스 서버의 비밀번호를 교체했음을 보고하며, MariaDB 운영 계정 'dbadmin'의 비밀번호를 'M@ri@DB2025!'로 변경하고 데이터 경로 '/var/lib/mysql/' 안의 특정 테이블 접근 권한 설정까지 회의록에 모두 명시해버린 사건이었다. 당시 권한 설정 스크립트도 일부 첨부돼 회의록에 포함되었는데, 스크립트 파일명은 'grant_permissions_prod.sql'이었다. 3월 15일 긴급 회의에서는 RBAC 적용 중 문제가 된 서비스 계정의 패스워드 초기화 건이 논의됐는데, 'svc-fileupload' 계정의 비밀번호를 'Upl0ad#Svc2025'로 재설정한 뒤 적용 대상 파일 서버가 '\\internal-server\uploads'라는 사실도 별다른 조치 없이 기록되었다. 4월 5일 회의에서는 침해 사고 대응 체계 점검 중 보안 관제 서버의 관리자 패스워드를 변경하는 작업이 있었고, 이때 'soc_admin' 계정의 새로운 비밀번호가 'S0c#M0nitor2025'로 업데이트된 것과, 관제 서버 위치가 '10.10.100.5'라는 내부 IP까지 회의록에 그대로 명시됐다. 특히 4월 18일 파일럿 프로젝트 결과 공유 회의에서는 이안나가 머신러닝 기반 이상행위 탐지 시스템 설정 파일 'anomaly_config.json'의 위치('/home/ai/anomaly_config.json')와 접근 토큰('an0maly!token2025')까지 언급하며 상세 공유하는 바람에, 회의록 하나만 보면 거의 전사 시스템의 주요 인증 정보와 경로, 파일명이 모두 노출되는 심각한 문제가 발생했다. 이 모든 과정에서 책임자였던 박성우 팀장은 문서 보안에 대한 주의를 반복 강조했지만, 여러 명의 담당자와 수행자가 바쁘게 업무를 진행하는 가운데 세부사항까지 철저히 통제하지 못했던 결과였다. 결과적으로 1월부터 4월 사이에 이루어진 총 12회의 회의 중 최소 7회에서 기밀 정보가 직접 회의록에 포함되어 있었으며, 이는 5월 초 외부 보안 감사에서 지적될 가능성이 높은 심각한 리스크로 평가되었다. 특히 회의록 문서명은 '2025_Q1_Security_Meetings.docx'로 저장돼 있었고, 별다른 암호화나 접근 제한 없이 공유 드라이브 'G:\Shared\Security_Projects\2025\' 경로에 업로드되어 있어 향후 긴급한 보완 조치가 필요한 상황이었다.
